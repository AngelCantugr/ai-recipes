## Purpose
Create clear, comprehensive documentation for code and systems

## When to Use
- Documenting new features
- Creating API documentation
- Writing technical specifications
- README files and getting started guides

## Template

```
I need to create documentation for [component/API/system]. Help me write clear, useful documentation.

What to Document:
[Code, API, system, or feature to document]

Audience:
- Primary readers: [developers, users, operators]
- Technical level: [beginner/intermediate/expert]
- What they're trying to accomplish: [use cases]

Documentation Type:
[ ] API Documentation
[ ] README/Getting Started
[ ] Architecture Documentation
[ ] User Guide
[ ] Troubleshooting Guide
[ ] Runbook/Operations Guide

Context:
[Paste code or describe the system]

Please help me create documentation with:

1. **Overview**
   - What it is and why it exists
   - Key concepts
   - When to use it

2. **Getting Started** (if applicable)
   - Prerequisites
   - Installation/setup
   - Quick example
   - First successful outcome

3. **Detailed Documentation**
   - API reference (inputs, outputs, errors)
   - Configuration options
   - Common patterns and examples
   - Best practices

4. **Troubleshooting**
   - Common issues and solutions
   - Error messages and meanings
   - How to get help

5. **Additional Sections** (as needed)
   - Architecture diagrams
   - Performance considerations
   - Security considerations
   - Migration guides
   - Contributing guidelines

Format:
- Use clear headings and structure
- Include code examples
- Add diagrams where helpful
- Keep language simple and direct
- Provide context for decisions
```

## Tips
- Write documentation as you code, not after
- Include examples for all common use cases
- Keep it up-to-date with code changes
- Test your documentation - can someone follow it?
- Start with "why" before "how"
- Use consistent terminology
- Link to related documentation
- Include version information
