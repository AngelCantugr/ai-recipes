# Technical Writing Polish ✍️

## Purpose
Improve clarity and effectiveness of technical communications

## When to Use
- Editing design documents
- Polishing RFCs or proposals
- Improving documentation
- Writing blog posts or announcements

## Template

```
I've written technical content that needs improvement. Help me make it clearer and more effective.

Content Type:
[ ] Design Document
[ ] RFC/Proposal
[ ] Technical Blog Post
[ ] Release Notes
[ ] Email/Announcement
[ ] Documentation
[ ] Architecture Decision Record

Audience:
[Who will read this?]

Goal:
[What do you want readers to understand or do?]

Current Draft:
[Paste your content here]

Please help me improve:

1. **Structure**
   - Is the flow logical?
   - Does it start with context/motivation?
   - Are sections well-organized?
   - Should anything be reordered?

2. **Clarity**
   - Identify confusing sentences
   - Suggest simpler alternatives
   - Point out jargon that needs definition
   - Flag assumptions that need explanation

3. **Completeness**
   - What questions will readers have?
   - What's missing?
   - Do examples cover key scenarios?
   - Are edge cases addressed?

4. **Conciseness**
   - Remove unnecessary words
   - Combine redundant sections
   - Tighten verbose explanations
   - Keep essential details

5. **Technical Accuracy**
   - Check for technical errors
   - Verify terminology usage
   - Ensure consistency

6. **Engagement**
   - Is the opening compelling?
   - Are transitions smooth?
   - Is the conclusion clear?
   - Does it maintain interest?

Specific concerns:
[Any particular areas you're unsure about?]
```

## Tips
- Read your writing out loud
- Use active voice
- Be specific, not vague
- Front-load important information
- Use headings and lists for scannability
- Include diagrams for complex concepts
- Get feedback before publishing
- Update based on common questions
